20. Core User Journeys

  1. New Patient Intake: Lead → AI draft response → Convert to patient → Send intake form → Patient completes at /portal/intake → Appointment booked → Insurance verified
  2. Clinical Visit: Check-in → Clinical notes (CDT template + AI assist) → X-rays (media upload) → Treatment plan presented → Proposal built (Good/Better/Best) → Cherry financing if needed → Patient e-signs at /proposal/[token]
  3. Billing Flow: Appointment → Insurance claim → Eligibility check → Payment (Square) → Invoice generated → Write-off tracking if needed
  4. AI Phone Call: Patient calls → Vapi AI handles → Transcript + summary stored → Staff reviews in Calls dashboard
  5. SEO Dashboard: Daily cron pulls PageSpeed scores → Weekly GSC sync → SEO auto-fix runs Mondays → Monthly SEO report generated and emailed
  6. No-Show Recovery: No-show detected every 30 min (Mon-Sat, 1-8pm) → Recovery message sent via SMS/email → Rebook tracked

23. Performance Considerations

  • 206 API routes — largest engine. Ensure Vercel function cold starts are acceptable.
  • 14 cron jobs — risk of Supabase connection pool exhaustion if crons fire simultaneously. Stagger execution times where possible.
  • Media uploads (Vercel Blob) — CDN-backed, fast for patient and staff access.
  • Recharts for analytics — SSR-safe config required.
  • Web vitals tracking active — use data to optimize LCP/CLS on patient-facing pages.
  • SEO health check daily — PageSpeed data available for continuous optimization.
  • Vapi transcripts — can be large; ensure /api/vapi route handles large payloads without timeout.
